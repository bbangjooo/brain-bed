## 1. 제품 개요

### 1.1 목적

Brain Bed는 AI 도구 과다 사용으로 인한 정신적 피로(Brain Fry)를 예방하기 위해, 일정 시간 연속 사용 시 컴퓨터 화면 자체를 명상 공간으로 전환하는 데스크탑 앱이다.

### 1.2 핵심 가치

"컴퓨터를 끄지 않아도, 컴퓨터 안에서 쉴 수 있다."

- 컴퓨터를 떠나지 못하는 사용자를 위해 화면 자체가 휴식 공간이 된다
- 시간 기반으로 "쉬어야 할 타이밍"을 객관적으로 알려준다
- 강제성이 있되, 불쾌하지 않은 경험(명상 + 클래식 음악 + 시각적 아름다움)을 제공한다

### 1.3 타겟 사용자

AI 도구(Claude Code, Codex CLI, Gemini CLI, Cursor 등)를 일상적으로 사용하며, 하루 수시간 이상 AI와 상호작용하는 개발자. 특히 1인 개발자 또는 소규모 팀에서 다양한 역할을 겸하는 개발자.

### 1.4 MVP 범위 요약

| 포함 | 미포함 (Phase 2 이후) |
|------|----------------------|
| 시간 기반 사용 감지 + 알림 | CLI 토큰 사용량 추적 |
| 전체화면 명상 모드 (Inner Peace Zone) | 3D UI (Three.js/Spline) |
| 정적 명상 UI (문구 + 그라데이션) | Audio Visualizer |
| 내장 클래식 음원 3~5곡 재생 | 통계 대시보드 |
| 시스템 트레이 상주 | 패턴 기반/복합 트리거 |
| 기본 설정 (시간, 간격) | Windows/Linux 지원 |

## 2. 기능 요구사항

### 2.1 시스템 트레이 앱

**설명**: 앱이 macOS 메뉴바에 상주하며, 현재 상태를 표시하고 주요 기능에 빠르게 접근할 수 있는 진입점을 제공한다.

**유저 스토리**:
- 사용자로서, 앱이 백그라운드에서 실행되면서 메뉴바에 아이콘으로 표시되길 원한다. 앱 창을 항상 열어두지 않아도 동작하게 하기 위해서다.
- 사용자로서, 트레이 아이콘을 클릭하면 현재 연속 사용 시간과 메뉴를 볼 수 있길 원한다. 지금 상태를 빠르게 확인하기 위해서다.

**우선순위**: P0 (필수)

**수용 기준**:
- [ ] 앱 실행 시 macOS 메뉴바에 트레이 아이콘이 표시된다
- [ ] 트레이 아이콘 클릭 시 드롭다운 메뉴가 나타난다
- [ ] 드롭다운 메뉴에 다음 항목이 포함된다:
  - 현재 연속 사용 시간 (예: "연속 사용: 45분")
  - "지금 쉬기" 버튼 (수동으로 Inner Peace Zone 진입)
  - "설정" 메뉴
  - "종료" 메뉴
- [ ] 트레이 아이콘 상태 표시:
  - 기본 상태: 기본 아이콘
  - 임계값 근접 (80% 이상): 경고 아이콘 (색상 변경)
  - 명상 중: 명상 상태 아이콘
- [ ] 앱 창(Dock 아이콘)은 숨기고, 트레이 아이콘으로만 접근 가능하다
- [ ] macOS 로그인 시 자동 시작 옵션을 지원한다

**상세 스펙**:
- Electron의 `Tray` API 사용
- 아이콘 크기: 16x16 (일반), 32x32 (@2x Retina)
- Template 이미지 사용 (macOS 다크모드 자동 대응)

### 2.2 사용 시간 추적

**설명**: 사용자의 컴퓨터 연속 사용 시간을 추적하고, 설정된 임계값을 초과하면 휴식 알림을 보낸다.

**유저 스토리**:
- 사용자로서, 내가 컴퓨터를 얼마나 연속으로 사용했는지 자동으로 추적되길 원한다. 내가 직접 시간을 재지 않아도 되게 하기 위해서다.
- 사용자로서, 90분 연속 사용 시 알림을 받고 싶다. 휴식할 타이밍을 놓치지 않기 위해서다.

**우선순위**: P0 (필수)

**수용 기준**:
- [ ] 앱 시작 시점부터 연속 사용 시간 카운트를 시작한다
- [ ] 시스템 유휴 상태(마우스/키보드 입력 없음)가 5분 이상 지속되면 타이머를 리셋한다
- [ ] 설정된 임계값(기본 90분) 도달 시 macOS 네이티브 알림을 발송한다
- [ ] 알림 내용: "90분 연속 사용 중이에요. 잠깐 쉬어볼까요?" + "쉬기" / "10분 뒤에" 버튼
- [ ] "10분 뒤에" 선택 시 10분 후 재알림한다
- [ ] 알림을 3회 연속 무시/연기하면 4번째 알림에서 강제 진입을 제안한다 (사전 동의 설정 시)
- [ ] Inner Peace Zone 완료 후 타이머가 0으로 리셋된다
- [ ] 시스템 잠금/슬립 시 타이머를 일시 정지한다

**상세 스펙**:
- Electron `powerMonitor` API로 시스템 유휴/잠금/슬립 감지
- 타이머 정밀도: 1분 단위 (초 단위 정밀도 불필요)
- 알림: Electron `Notification` API (macOS 네이티브 알림 센터 연동)
- 유휴 감지 임계값: 5분 (설정 불가, 하드코딩)
- 타이머 상태는 메모리에서 관리 (SQLite 저장 불필요, 앱 재시작 시 리셋)

**데이터 흐름**:
```
[앱 시작] → 타이머 시작 → [유휴 감지] → 타이머 일시정지/리셋
                          ↓
              [임계값 도달] → 알림 발송
                              ↓
                    [사용자 응답] → "쉬기": Inner Peace Zone 진입
                                 → "나중에": 재알림 예약
                                 → 무시: 반복 카운트 증가
```

### 2.3 Inner Peace Zone (전체화면 명상 모드)

**설명**: 사용자가 휴식 알림을 수락하면 진입하는 전체화면 강제 휴식 모드. 키보드 입력을 차단하고, 설정된 시간 동안 명상 환경을 제공한다.

**유저 스토리**:
- 사용자로서, 전체화면이 명상 공간으로 바뀌길 원한다. 다른 작업의 유혹 없이 온전히 쉴 수 있게 하기 위해서다.
- 사용자로서, 키보드 입력이 차단되길 원한다. 무의식적으로 코드를 치려는 습관을 막기 위해서다.
- 사용자로서, 긴급할 때는 특정 단축키로 빠져나올 수 있길 원한다. 정말 급한 상황에 대응하기 위해서다.

**우선순위**: P0 (필수)

**수용 기준**:
- [ ] 진입 시 전체화면으로 전환된다 (모든 다른 창 위에 표시)
- [ ] 진입 애니메이션: 0.8초에 걸쳐 페이드인 전환
- [ ] 일반 키보드 입력이 차단된다 (문자, 숫자, 특수문자, 기능키)
- [ ] 다음 입력은 허용된다:
  - 볼륨 조절 키 (F11/F12 또는 미디어 키)
  - 긴급 해제 핫키: `Cmd + Shift + Esc` (기본값, 설정 변경 가능)
- [ ] 타이머가 화면에 표시되며, 남은 시간을 시각적으로 보여준다
- [ ] 타이머 완료 시 "수고했어요" 메시지와 함께 3초 후 자동 해제된다
- [ ] 긴급 해제 핫키 입력 시 확인 다이얼로그가 표시된다:
  - "정말 나가시겠어요? 아직 X분 남았어요."
  - "계속 쉬기" / "나가기" 버튼
  - "나가기" 선택 시 전체화면 해제
- [ ] `Cmd+Q`, `Cmd+W`, `Cmd+Tab`, `Alt+Tab` 등 앱 전환/종료 단축키가 차단된다
- [ ] Mission Control, Spotlight 등 시스템 단축키 차단 (접근성 권한 필요)

**상세 스펙**:
- Electron `BrowserWindow` 옵션:
  - `fullscreen: true`
  - `alwaysOnTop: true`
  - `closable: false` (명상 중)
  - `kiosk: true` (macOS에서 전체화면 + 메뉴바 숨김)
- 키보드 차단 구현 방식:
  - 1차: Electron `globalShortcut` API로 주요 단축키 등록 차단
  - 2차: Renderer 프로세스에서 `keydown` 이벤트 `preventDefault()`
  - 3차 (필요시): macOS 접근성 API를 통한 네이티브 키보드 이벤트 가로채기 (`node-addon-api` 또는 `CGEventTap`)
- 명상 시간 옵션: 5분, 10분(기본), 15분, 20분, 30분
- 진입 전 명상 시간 선택 UI 제공 (알림에서 또는 진입 직후 3초간)

**키보드 차단 구현 우선순위**:
```
Level 1 (Electron 내장 기능만으로 구현):
  - globalShortcut으로 Cmd+Q, Cmd+W, Cmd+Tab 등록
  - BrowserWindow kiosk 모드
  - renderer에서 keydown preventDefault
  → 대부분의 일반 키 입력 차단 가능
  → 일부 시스템 레벨 단축키는 차단 불가

Level 2 (접근성 권한 활용):
  - macOS Accessibility API 사용
  - CGEventTap으로 시스템 레벨 키보드 이벤트 가로채기
  - node-addon-api로 네이티브 모듈 작성
  → Mission Control, Spotlight 등까지 차단 가능
  → 접근성 권한 요청 필요
```

MVP에서는 Level 1을 기본으로 구현하고, Level 2는 사용자 피드백에 따라 점진적으로 추가한다.

### 2.4 명상 UI

**설명**: Inner Peace Zone 진입 시 표시되는 시각적 명상 환경. 동기부여 문구, 그라데이션 배경, 타이머 시각화로 구성된다.

**유저 스토리**:
- 사용자로서, 아름다운 시각적 환경을 보며 쉬고 싶다. 단순한 검은 화면이 아니라 쉬고 싶은 마음이 드는 공간이길 원하기 때문이다.
- 사용자로서, 동기부여 문구를 보며 휴식의 가치를 되새기고 싶다. 휴식을 스킵하지 않도록 마음을 다잡기 위해서다.

**우선순위**: P0 (필수)

**수용 기준**:
- [ ] 전체화면에 부드러운 그라데이션 배경이 표시된다
- [ ] 그라데이션이 천천히 색상이 변한다 (10초에 걸쳐 전환, 루프)
- [ ] 화면 중앙에 동기부여 문구가 표시된다
- [ ] 문구는 30초마다 페이드 전환으로 다음 문구로 바뀐다
- [ ] 최소 20개 이상의 문구가 준비되어 있다
- [ ] 타이머가 시각적으로 표시된다:
  - 원형 프로그레스 바 (남은 시간 비율)
  - 디지털 타이머 텍스트 (MM:SS 형식)
  - 화면 하단 또는 모서리에 배치 (시선을 뺏지 않도록)
- [ ] 전체적인 폰트는 세리프 또는 라운드 산세리프로 부드러운 느낌
- [ ] 화면 전환 시 모든 요소가 부드럽게 애니메이션된다

**상세 스펙**:

그라데이션 배경:
- CSS `linear-gradient` + `animation`으로 구현
- 색상 팔레트 예시 (차분한 톤):
  - 세트 1: `#0f0c29` → `#302b63` → `#24243e` (딥 퍼플)
  - 세트 2: `#0d1b2a` → `#1b2838` → `#2c3e50` (딥 블루)
  - 세트 3: `#1a1a2e` → `#16213e` → `#0f3460` (네이비)
  - 세트 4: `#1b1b1b` → `#2d2d2d` → `#1b1b1b` (다크 뉴트럴)
- `background-size: 400% 400%` + `animation: gradient 10s ease infinite`

동기부여 문구 예시:
- "잠깐 멈추는 것도 전진이에요."
- "뇌도 충전이 필요합니다."
- "지금 이 순간, 아무것도 하지 않아도 괜찮아요."
- "좋은 코드는 쉬고 난 뒤에 나옵니다."
- "숨을 깊이 들이쉬고, 천천히 내쉬세요."
- "AI는 기다려줍니다. 당신의 뇌는 안 기다려줘요."
- (추가 문구는 별도 JSON 파일에서 관리)

타이머 시각화:
- SVG 원형 프로그레스 바
- `stroke-dasharray` + `stroke-dashoffset` 애니메이션
- 반지름: 60px, 선 두께: 4px
- 색상: 반투명 화이트 (`rgba(255, 255, 255, 0.3)` 배경, `rgba(255, 255, 255, 0.8)` 진행)

### 2.5 음악 플레이어

**설명**: Inner Peace Zone에서 퍼블릭 도메인 클래식 음원을 재생하여 명상 분위기를 조성한다.

**유저 스토리**:
- 사용자로서, 명상 중에 클래식 음악이 자동으로 재생되길 원한다. 시각적 경험과 함께 청각적으로도 편안해지기 위해서다.
- 사용자로서, 볼륨을 조절하거나 음악을 끌 수 있길 원한다. 상황에 따라 소리 없이 쉬고 싶을 수도 있기 때문이다.

**우선순위**: P1 (중요)

**수용 기준**:
- [ ] Inner Peace Zone 진입 시 음악이 자동 재생된다 (페이드인 2초)
- [ ] 내장 클래식 음원 3~5곡이 앱에 포함된다
- [ ] 모든 음원은 퍼블릭 도메인이다 (저작권 이슈 없음)
- [ ] 재생 중인 곡 제목과 작곡가가 화면 하단에 작게 표시된다
- [ ] 볼륨 조절 가능 (키보드 볼륨 키 또는 화면 내 슬라이더)
- [ ] 음소거 토글 가능
- [ ] 곡이 끝나면 다음 곡 자동 재생 (셔플)
- [ ] Inner Peace Zone 종료 시 음악이 페이드아웃(2초) 후 정지한다
- [ ] 설정에서 자동 재생 켜기/끄기 옵션 제공

**상세 스펙**:

음원 형식:
- 형식: MP3 (128kbps 이상) 또는 OGG
- 곡당 최대 용량: 10MB
- 총 내장 음원 용량: 50MB 이내

음원 후보 목록 (퍼블릭 도메인):
| 곡명 | 작곡가 | 출처 |
|------|--------|------|
| Gymnopedies No.1 | Erik Satie | Musopen/IMSLP |
| Clair de Lune | Claude Debussy | Musopen/IMSLP |
| Prelude in E minor Op.28 No.4 | Frederic Chopin | Musopen/IMSLP |
| Air on the G String | J.S. Bach | Musopen/IMSLP |
| Nocturne Op.9 No.2 | Frederic Chopin | Musopen/IMSLP |

구현:
- Web Audio API (`AudioContext`) 사용
- `GainNode`로 볼륨 제어 및 페이드인/아웃
- 음원 파일은 앱 번들의 `resources/audio/` 디렉토리에 포함
- 재생 상태 관리: React state (`isPlaying`, `currentTrack`, `volume`)

### 2.6 설정 화면

**설명**: 사용자가 앱의 핵심 동작을 커스터마이즈할 수 있는 설정 화면.

**유저 스토리**:
- 사용자로서, 알림 간격과 명상 시간을 내 업무 패턴에 맞게 조절하고 싶다. 너무 잦거나 너무 긴 명상은 오히려 방해가 되기 때문이다.
- 사용자로서, 긴급 해제 핫키를 변경하고 싶다. 기존에 사용하는 단축키와 충돌할 수 있기 때문이다.

**우선순위**: P1 (중요)

**수용 기준**:
- [ ] 트레이 메뉴에서 "설정"을 클릭하면 설정 창이 열린다
- [ ] 설정 창은 독립된 작은 윈도우로 표시된다 (400x500px)
- [ ] 다음 설정 항목을 제공한다:

| 설정 항목 | 타입 | 기본값 | 범위 |
|-----------|------|--------|------|
| 연속 사용 알림 시간 | 슬라이더 + 숫자 입력 | 90분 | 30~180분 (10분 단위) |
| 재알림 간격 | 드롭다운 | 10분 | 5분, 10분, 15분, 30분 |
| 기본 명상 시간 | 드롭다운 | 10분 | 5분, 10분, 15분, 20분, 30분 |
| 긴급 해제 핫키 | 키 입력 캡처 | Cmd+Shift+Esc | 사용자 지정 |
| 강제 진입 모드 | 토글 | 꺼짐 | 켜짐/꺼짐 |
| 음악 자동 재생 | 토글 | 켜짐 | 켜짐/꺼짐 |
| 로그인 시 자동 시작 | 토글 | 꺼짐 | 켜짐/꺼짐 |

- [ ] 설정 변경 시 즉시 적용된다 (저장 버튼 없음)
- [ ] 설정값은 SQLite에 영구 저장된다
- [ ] 앱 재시작 시 마지막 설정값이 복원된다

**상세 스펙**:
- Electron `BrowserWindow` (별도 윈도우)
- React 컴포넌트로 구현
- 설정 변경 시 Main Process에 IPC로 전달
- `electron-store` 또는 직접 SQLite 사용

## 3. 비기능 요구사항

### 3.1 성능

| 항목 | 기준 |
|------|------|
| 앱 시작 시간 | 3초 이내 (트레이 아이콘 표시까지) |
| 메모리 사용량 (유휴 시) | 100MB 이하 |
| 메모리 사용량 (명상 모드) | 200MB 이하 |
| CPU 사용량 (유휴 시) | 1% 이하 |
| CPU 사용량 (명상 모드) | 5% 이하 |
| Inner Peace Zone 진입 시간 | 1초 이내 (알림 수락부터 전체화면까지) |
| 타이머 정확도 | 1분당 오차 1초 이내 |

### 3.2 안정성

- 앱 크래시 시 자동 재시작 (Electron `app.relaunch()`)
- 명상 모드 중 크래시 시 안전하게 전체화면 해제
- 타이머 상태는 메모리에서 관리 (크래시 시 리셋 허용)

### 3.3 보안 및 개인정보

- 모든 데이터는 로컬에만 저장된다 (서버 전송 없음)
- 네트워크 요청 없음 (MVP에서는 외부 통신 완전 차단)
- 접근성 권한 요청 시 사용 목적을 명확히 고지한다
- SQLite DB 파일은 앱 전용 디렉토리에 저장 (`~/Library/Application Support/BrainBed/`)

### 3.4 접근성

- macOS 접근성 권한 요청 흐름:
  1. 최초 실행 시 권한 필요 여부 확인
  2. 권한이 없으면 안내 다이얼로그 표시 (왜 필요한지 설명)
  3. 시스템 설정 > 개인정보 및 보안 > 접근성으로 이동 유도
  4. 권한 없이도 Level 1 키보드 차단은 동작 (Level 2만 비활성화)
- 명상 UI의 텍스트 대비율: WCAG AA 기준 충족 (4.5:1 이상)

### 3.5 배포

- 배포 도구: `electron-builder`
- 설치 파일: DMG (macOS)
- 코드 서명: Apple Developer 인증서 (추후, MVP에서는 미서명 허용)
- 자동 업데이트: MVP에서 미지원 (수동 업데이트)

| `settings:changed` | Main → Renderer | `{ key: string, value: any }` | 설정 변경 알림 |
| `audio:control` | Renderer → Main | `{ action: string }` | 음악 재생 제어 (필요 시) |

```sql
INSERT INTO settings (key, value) VALUES

-- 명상 세션 기록 (통계용, MVP에서는 기본 기록만)
CREATE TABLE meditation_sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    started_at DATETIME NOT NULL,
    ended_at DATETIME,
    duration_seconds INTEGER NOT NULL,       -- 설정된 명상 시간
    actual_duration_seconds INTEGER,          -- 실제 명상 시간
    completed BOOLEAN DEFAULT FALSE,          -- 정상 완료 여부
    trigger_type TEXT DEFAULT 'manual',       -- 'auto' | 'manual'
    continuous_usage_minutes INTEGER,         -- 진입 시점의 연속 사용 시간
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 알림 기록
CREATE TABLE notification_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sent_at DATETIME NOT NULL,
    response TEXT,                            -- 'accepted' | 'postponed' | 'ignored'

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

| Electron + React + TypeScript 프로젝트 초기화 | 빌드 가능한 빈 앱 |
| Tailwind CSS 설정 | 스타일 시스템 |
| 시스템 트레이 아이콘 + 드롭다운 메뉴 구현 | TrayManager 모듈 |
| 트레이 아이콘 상태 변경 로직 | 아이콘 에셋 + 상태 전환 |

**완료 기준**: 앱 실행 시 메뉴바에 아이콘이 표시되고, 클릭 시 드롭다운 메뉴가 동작한다.

### Week 2: 사용 시간 추적 + 알림

| 작업 | 산출물 |
|------|--------|
| ActivityTracker 모듈 구현 | 타이머 + 유휴 감지 |

**완료 기준**: 설정한 시간(기본 90분)이 되면 알림이 뜨고, "쉬기"를 누르면 다음 단계로 넘어간다.

### Week 3: Inner Peace Zone + 명상 UI

| 작업 | 산출물 |
|------|--------|

| 명상 시간 선택 UI | 시간 선택 화면 |
| 명상 완료 화면 | 완료 + 자동 해제 |

**완료 기준**: 전체화면 명상 모드에 진입할 수 있고, 타이머가 동작하며, 시간이 끝나면 자동으로 해제된다.

### Week 4: 키보드 차단 + 음악 플레이어

| 볼륨 조절 + 페이드인/아웃 | 오디오 제어 UI |

**완료 기준**: 명상 모드에서 키보드가 차단되고, 음악이 재생되며, 긴급 해제가 동작한다.

### Week 5: 설정 + 데이터 저장

**완료 기준**: 설정을 변경하면 앱 동작에 반영되고, 재시작 후에도 설정이 유지된다.

**완료 기준**: DMG 파일로 설치 가능하고, 전체 플로우(트레이 → 알림 → 명상 → 해제)가 안정적으로 동작한다.

## 8. 리스크 관리 계획

### 8.1 기술적 리스크

| 리스크 | 심각도 | 확률 | 대응 전략 |
|--------|--------|------|-----------|
| **macOS 키보드 차단 불완전** | 높음 | 높음 | Level 1(Electron 내장)로 MVP 출시. 사용자가 의도적으로 빠져나가는 것은 허용. Level 2(네이티브 모듈)는 Phase 2에서 대응. "완벽한 차단"이 아니라 "차단 의도"에 집중. |
| **접근성 권한 거부** | 중간 | 중간 | 접근성 권한 없이도 Level 1 차단은 동작하도록 설계. 권한 안내 UI를 친절하게 작성. 권한 없을 때의 동작을 명확히 안내. |
| **Electron 메모리 사용량** | 중간 | 중간 | 유휴 시 불필요한 렌더러 프로세스 종료. 명상 윈도우는 필요할 때만 생성. 메모리 프로파일링 주기적 실행. |
| **Web Audio API 호환성** | 낮음 | 낮음 | Electron이 Chromium 기반이므로 호환성 문제 거의 없음. Fallback으로 HTML5 Audio 태그 사용 가능. |
| **kiosk 모드에서 시스템 단축키** | 중간 | 높음 | kiosk 모드가 모든 시스템 단축키를 차단하지 못할 수 있음. 사용자에게 "명상 모드에서는 다른 작업을 자제해주세요" 안내. |

### 8.2 사용자 경험 리스크

| 리스크 | 심각도 | 확률 | 대응 전략 |
|--------|--------|------|-----------|
| **강제 차단 거부감** | 중간 | 중간 | MVP에서 강제 진입은 기본 꺼짐. 긴급 해제 핫키 항상 제공. 첫 사용 시 온보딩에서 앱 철학 설명. |
| **알림 무시 습관** | 중간 | 높음 | 재알림 간격 설정 제공. 3회 무시 후 강제 진입 제안 (선택적). 트레이 아이콘 색상 변경으로 시각적 리마인더. |
| **명상 시간이 너무 길다/짧다** | 낮음 | 중간 | 5/10/15/20/30분 다양한 옵션 제공. 진입 시 시간 선택 가능. 기본값을 설정에서 변경 가능. |

### 8.3 리스크 모니터링

- Week 1 종료 시점: 트레이 앱 + Electron 기본 동작 검증
- Week 3 종료 시점: 키보드 차단 Level 1 동작 검증 (이 시점에서 Level 2 필요 여부 판단)
- Week 5 종료 시점: 전체 플로우 동작 검증 + 성능 측정
