# Brain Bed - Idea Brief

> AI 시대의 강제 휴식 데스크탑 앱

---

## 1. 프로젝트 개요

**한 줄 요약**: AI 도구 과다 사용으로 인한 정신적 피로(Brain Fry)를 감지하고, 컴퓨터 안에서 강제 명상 공간을 제공하는 Electron 기반 데스크탑 앱.

**프로젝트명**: Brain Bed

**형태**: Electron 데스크탑 앱 (macOS 우선, 추후 Windows/Linux)

---

## 2. 문제 정의

### 핵심 문제

AI 도구(Claude Code, Codex, Gemini CLI 등)의 등장으로 개발자들의 작업 강도가 급격히 높아졌다. AI와 "지능 대 지능"으로 대화하며 작업하는 과정은 기존 코딩보다 훨씬 높은 인지 부하를 발생시킨다. 이로 인해 발생하는 증상이 **Brain Fry** -- 과도한 AI 도구 사용으로 인한 심각한 정신적 피로감과 번아웃 현상이다.

### 문제의 구조

1. **인지 과부하**: AI의 출력을 이해하고 판단하는 작업이 연속적으로 발생하며, 뇌가 처리할 수 있는 한계를 초과
2. **중독성**: 생산성이 극대화되는 느낌에 컴퓨터를 벗어나지 못하는 중독 상태 발생
3. **휴식 불능**: "조금만 더" 심리로 자발적 휴식이 거의 불가능
4. **자각 부재**: 본인이 피로 상태임을 인지하지 못하거나, 인지하더라도 멈추지 못함

### 문제를 겪는 상황

- Claude Code로 몇 시간째 연속 작업 중일 때
- AI와의 대화에서 맥락을 유지하려고 과도하게 집중할 때
- 토큰 사용량이 급증하는데도 멈추지 못할 때
- 작업이 끝났는데도 "한 가지만 더" 하려고 할 때

---

## 3. 타겟 유저

### 주요 타겟

**AI 도구 헤비유저 개발자**

- Claude Code, GitHub Copilot, Cursor, Codex CLI, Gemini CLI 등을 일상적으로 사용
- 하루 수시간 이상 AI와 상호작용하며 코딩
- 1인 개발자 또는 소규모 팀에서 다양한 역할을 겸하는 개발자
- Brain Fry 증상을 경험했거나 경험 중

### 보조 타겟

- AI 기반 글쓰기/디자인 도구를 장시간 사용하는 크리에이터
- ChatGPT 등 대화형 AI를 업무에 과도하게 활용하는 지식 노동자

### 유저 페르소나

> 30대 1인 개발자. Claude Code로 하루 8시간 이상 작업. 생산성은 높아졌지만 저녁이면 머리가 멍하고, 주말에도 회복이 안 되는 느낌. 쉬어야 한다는 건 알지만, AI가 바로 답을 주니까 "이것만 더" 하다가 결국 번아웃.

## 4. 핵심 가치 제안 (UVP)

**"컴퓨터를 끄지 않아도, 컴퓨터 안에서 쉴 수 있다."**

- 휴식을 위해 컴퓨터를 떠날 필요 없음 (중독 상태의 유저는 어차피 떠나지 못함)
- 컴퓨터 화면 자체가 명상 공간으로 전환
- AI 사용량 데이터 기반으로 "쉬어야 할 타이밍"을 객관적으로 알려줌
- 강제성이 있되, 불쾌하지 않은 경험 (명상 + 클래식 음악 + 아름다운 3D UI)

---

## 5. 주요 기능

### 5.1 AI 토큰 사용량 추적

CLI 기반 AI 도구의 활동을 모니터링하여 사용량을 추적한다.

**추적 대상**:
- Claude Code (API 토큰 사용량 또는 세션 시간)
- Codex CLI
- Gemini CLI
- 추후 확장 가능한 플러그인 구조

**Notification 트리거 기준 (제안)**:
- **시간 기반**: 연속 AI 사용 시간 임계값 초과 (예: 90분 연속 사용)
- **토큰 기반**: 누적 토큰 사용량 임계값 초과 (예: 일일 100K 토큰)
- **패턴 기반**: 짧은 시간 내 고밀도 상호작용 감지 (예: 10분 내 20회 이상 프롬프트)
- **복합 기준**: 위 조건들의 가중 조합 (사용자 커스터마이즈 가능)
- **시간대 기반**: 늦은 시간(예: 밤 10시 이후) 사용 시 더 낮은 임계값 적용

### 5.2 Inner Peace Zone

Notification 수락 시 진입하는 강제 휴식 모드.

- 화면 전체를 점령하는 몰입형 UI
- 모든 키보드 입력 차단 (특정 해제 핫키 제외)
- 기본 10분, 사용자 선택 가능 (5분/10분/15분/20분/30분)
- 타이머 표시 (남은 시간 시각화)
- 중간 해제 시 "정말 나가시겠습니까?" 확인 + 쿨다운 설정 가능

### 5.3 명상 UI/UX

- **3D 비주얼**: Spline 커뮤니티 스타일의 3D 오브젝트와 동기부여 문구
- **Distorting Typography**: 움직이는 타이포그래피로 명상적 분위기 연출
- **Audio Visualizer**: 재생 중인 음악에 반응하는 시각적 이펙트
- **미니멀 인터랙션**: 마우스 움직임에 반응하는 파티클/오브젝트 (키보드는 차단)

### 5.4 클래식 음악 플레이어

- 퍼블릭 도메인 클래식 음원 사용 (IMSLP, Musopen)
- 명상/집중에 적합한 큐레이션 플레이리스트
- 장르별 분류: 피아노 독주, 현악, 관현악 등
- 셔플/반복 재생
- 볼륨 컨트롤 (키보드 차단 상태에서도 볼륨 조절 가능)

### 5.5 통계 대시보드

- 일별/주별 AI 사용량 추이
- 명상 횟수 및 총 명상 시간
- Brain Fry 지수 (사용 패턴 기반 추정)
- 연속 명상 스트릭

## 6. MVP 범위

### MVP에 포함 (Phase 1)

| 기능 | 설명 |
|------|------|
| 시간 기반 트리거 | 연속 사용 시간 기반 notification (토큰 추적 없이 타이머만) |
| 기본 Inner Peace Zone | 전체 화면 모드 + 키보드 차단 + 타이머 |
| 정적 명상 UI | 간단한 동기부여 문구 + 그라데이션 배경 (3D는 후순위) |
| 음악 재생 | 내장 클래식 음원 3~5곡 재생 |
| 시스템 트레이 | 메뉴바/시스템 트레이 상주 |
| 기본 설정 | 명상 시간, 알림 간격 설정 |

### Phase 2 (Post-MVP)

- CLI 토큰 사용량 실제 추적 (Claude Code 로그 파싱 등)
- 3D UI (Three.js/Spline 연동)
- Audio Visualizer
- 통계 대시보드
- 패턴 기반 / 복합 트리거
- 시간대별 임계값 조정

### Phase 3 (확장)

- Windows/Linux 지원
- 플러그인 시스템 (추적 대상 확장)
- 커뮤니티 플레이리스트
- AI 사용 패턴 분석 리포트

## 7. 차별점

### 기존 솔루션과의 비교

| 솔루션 | 접근 방식 | Brain Bed과의 차이 |
|--------|----------|-------------------|
| **Screen Time (macOS)** | 앱별 사용 시간 제한 | AI 도구 특화 아님, 단순 차단만 제공 |
| **Freedom / Cold Turkey** | 웹사이트/앱 차단 | CLI 도구는 차단 불가, 휴식 경험 없음 |
| **Pomodoro 앱들** | 정해진 간격으로 휴식 | AI 사용량 기반이 아닌 기계적 타이머 |
| **명상 앱 (Calm, Headspace)** | 별도 앱에서 명상 | 컴퓨터 작업 중 전환 불편, 강제성 없음 |

### Brain Bed만의 차별점

1. **AI 도구 사용량 인식**: 단순 시간이 아닌, AI 도구 사용 강도 기반 휴식 제안
2. **컴퓨터 내 휴식**: 컴퓨터를 떠나지 않고도 쉴 수 있는 공간 제공
3. **강제성 + 심미성**: 차단만 하는 것이 아니라, 쉬고 싶어지는 환경 조성
4. **개발자 특화**: 터미널/CLI 기반 AI 도구를 이해하는 유일한 휴식 앱

## 8. 리스크 및 고려사항

### 기술적 리스크

| 리스크 | 심각도 | 대응 방안 |
|--------|--------|----------|
| 키보드 입력 차단의 OS 레벨 구현 | 높음 | Electron의 globalShortcut + 접근성 API 활용, OS별 네이티브 모듈 필요 가능성 |
| CLI 토큰 사용량 추적의 기술적 난이도 | 중간 | MVP에서는 시간 기반으로 시작, 로그 파일 파싱으로 점진적 구현 |
| macOS 보안 정책 (키보드 차단) | 높음 | 접근성 권한 요청 필요, 사용자 신뢰 확보 중요 |
| 3D 렌더링 성능 | 낮음 | MVP에서 제외, Phase 2에서 성능 테스트 후 적용 |

### 사용자 경험 리스크

| 리스크 | 심각도 | 대응 방안 |
|--------|--------|----------|
| 강제 차단에 대한 거부감 | 중간 | 긴급 해제 핫키 제공, 점진적 차단 (처음엔 부드럽게) |
| "또 다른 생산성 앱" 피로감 | 중간 | 심미적 경험 강조, 최소한의 설정으로 시작 |
| Notification 무시 습관화 | 중간 | 반복 알림 + 단계적 강도 증가 옵션 |

### 비즈니스 고려사항

- **수익 모델**: 기본 무료 + 프리미엄(추가 음원, 고급 통계, 커스텀 테마)
- **음원 저작권**: 퍼블릭 도메인(IMSLP, Musopen) 사용으로 저작권 이슈 회피
- **개인정보**: 토큰 사용량 등 민감 데이터는 로컬 저장 원칙, 서버 전송 없음

## 부록: Notification 트리거 기준 상세

MVP에서는 아래 기본값으로 시작하고, 사용자 피드백을 통해 조정한다.

```
기본 설정:
  연속_사용_시간: 90분
  알림_반복_간격: 30분 (첫 알림 무시 시)
  야간_모드_시작: 22:00
  야간_임계값_배수: 0.7 (낮 기준의 70%)
```

사용자가 알림을 3회 연속 무시할 경우, 4번째부터는 "강제 진입 모드"를 제안한다 (사용자가 사전에 동의한 경우에만).
